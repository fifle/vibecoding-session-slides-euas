SPEC-DRIVEN DEVELOPMENT (SDD)

1. Spec

Written description of what you want to build.

2. Acceptance Criteria

How do you know it's ready? = tests

3. Task Breakdown

Small, tested steps.

4. Iterate

Feedback → change spec → new iteration.

Why SDD + AI?

  • Clarity is key
    LLM performs better when you are precise.
  • Spec = Guide
    Not just "do something", 
    but a concrete plan.
  • Criteria = Test
    Not "show me", 
    but "does it work like this?".
